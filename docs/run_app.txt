###################### OPTIONAL ARGUMENTS ######################

You can run the program with two optional arguments:

    - debug:    prints debug messages on the console
    - logging:  write logs to log files

To turn on 'debug mode' use '-d' or '--debug' options
To turn on 'logging mode' use '-l' or '--log' options

If you don't pass any of these options, the program assumes debug and
logging mode are off.

###################### GET USAGE HELP ######################

To get help about program usage use '-h' or '--help' options.

If you run the program with help option it prompts a usage string
and stops because it assumes you don't know how to run the program.

###################### MANDATED ARGUMENTS ######################

You must pass to the program:

    - personality:          used to identify which bot run
    - api address number:   used to identify which API connects to

A valid 'personality' is one within the 'personality' section defined in the 'sample/resources.json'.
A valid 'api address number' is one within the 'api_address' section define in the 'sample/resources.json'.

Please, read the 'docs/resources_setup.txt' for more information about 'personality' section.

###################### VALID COMMAND EXAMPLES ######################

python -m airquality personality api_address_number
python -m airquality --debug personality api_address_number
python -m airquality --log -d personality api_address_number
python -m airquality --help

###################### INVALID COMMAND EXAMPLES ######################

python -m airquality --debug personality
python -m airquality --log -d api_address_number
python -m airquality api_address_number
